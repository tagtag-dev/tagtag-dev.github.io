Quick Start

Get up and running with Tagtag in minutes.

This guide will help you set up the Tagtag backend services and frontend application on your local machine. Don't worry, we'll walk you through the entire process step by step, as simple as teaching a child to walk.

Prerequisites

Ensure you have the following software installed. If not, go install them now, don't delay.

Backend

  • Java JDK 17+ - Yes, we're using the latest Java, keeping up with the times
  • Maven 3.8+ - A great helper for dependency management, no need to manually download jar files anymore
  • MySQL 8.0+ - The cornerstone of data storage, all data depends on it
  • Redis 6.0+ - A caching accelerator that makes the system fly

Frontend

  • Node.js >= 20.12.0 - The foundation for frontend runtime
  • pnpm >= 10.0.0 - A faster package manager than npm, everyone who uses it loves it

Installation

1. Clone the repository

git clone https://github.com/tagtag-dev/tagtag-starter.git
cd tagtag-starter

This step shouldn't be a problem, right? If there are issues, check your network connection or try a different approach.

2. Initialize Database (Backend)

  1. Create Database: Create a MySQL database named tagtag.
    CREATE DATABASE tagtag CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

    Remember, the database name must be tagtag, don't change it to something else, or the system won't be able to find the database.
  2. Automatic Initialization: Tagtag Starter uses Spring Boot's automatic initialization feature, which executes database scripts automatically when the application starts. Yes, it's that smart, no need to manually import SQL files anymore.
    Script Locations:
    • Table structure scripts: src/main/resources/db/schema.sql under each module
    • Initial data scripts: src/main/resources/db/data/ directory under each module

    Auto-loading Configuration (already configured in backend/tagtag-start/src/main/resources/application-dev.yml):
    spring:
  datasource:
    init:
      mode: always
      schema-locations:
        - classpath*:db/schema.sql
      data-locations:
        - classpath*:db/data/**.sql
  3. Initial Data Description:
    • Admin account: admin / admin123 - Super administrator with all permissions
    • Normal user account: user / user123 - Normal user with limited permissions
    • Initial roles: Super Admin, Admin, Normal User - Permission levels, each with their own responsibilities
    • Initial menus: System Management, User Management, Role Management, Menu Management, etc. - Everything you need is there

    Remember these initial account passwords, don't come back to us later when you can't log in.

3. Configure Application (Backend)

Edit backend/tagtag-start/src/main/resources/application.yml to match your local environment (Database credentials, Redis host).

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/tagtag?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
  data:
    redis:
      host: localhost
      port: 6379

Replace your_password with your actual database password. Don't use the default password, or don't blame us if hackers break in.

4. Start Backend Service

cd backend
mvn clean package -DskipTests
java -jar tagtag-start/target/tagtag-start-0.0.1-SNAPSHOT.jar

Alternatively, run TagtagApplication.java directly from your IDE. If you're using IDEA, just right-click and run, we don't need to teach you that, right?

Wait a moment, see a bunch of logs in the console, and finally see something like "Started TagtagApplication in X.XXX seconds", which means the startup was successful. Congratulations, the backend service is running!

5. Start Frontend Application

  1. Enter the frontend directory:
    cd frontend
  2. Install dependencies:
    pnpm install

This step may take a while, after all, it needs to download so many dependency packages. Be patient, don't rush, good things come to those who wait.

  1. Start the development server: 
    pnpm run dev

See something like "Local: http://localhost:5666/", which means the frontend has also started successfully. Great, both frontend and backend are running!

Verification

After successful startup, you can access the following addresses to verify the system is running normally:

  • Frontend Application: Access http://localhost:5666 - Login interface, try logging in with admin/admin123
  • Backend API Docs: Access http://localhost:8080/index.html (Scalar) - See what API interfaces the system provides
  • Health Check: Access http://localhost:8080/actuator/health - Check if the backend service is healthy

If these addresses can be accessed normally, congratulations, you have successfully set up the Tagtag Starter system! Give yourself a round of applause!

Common Issues and Solutions

If you unfortunately encounter problems, don't panic, we've prepared a list of common issues, see if it can help you.

1. Database Connection Failure

Problem Description: Database connection error when starting the application.

Solutions:

  • Check if MySQL service is running normally - First confirm the service isn't down
  • Verify database name, username, and password are correct - Don't type them wrong
  • Check if database address and port are correct - Default is localhost:3306
  • Confirm the database user has sufficient permissions - Give it enough permissions, don't be too stingy

2. Redis Connection Failure

Problem Description: Redis connection error when starting the application.

Solutions:

  • Check if Redis service is running normally - Confirm Redis isn't down
  • Verify Redis address and port are correct - Default is localhost:6379
  • Check Redis password (if set) - Don't forget the password
  • Confirm Redis instance allows remote connections - Check firewall settings

3. Frontend Cannot Access Backend API

Problem Description: Frontend application cannot connect to backend API.

Solutions:

  • Check if backend service is running normally - If the backend isn't running, the frontend can't access it
  • Verify the API address configured in the frontend is correct - Check the configuration file
  • Check CORS configuration - Cross-origin issues are a common pitfall
  • Check firewall settings to ensure ports are not blocked - Don't let the firewall block your way

4. Port Conflict When Starting

Problem Description: Port already in use error when starting the application.

Solutions:

  • Modify backend port: Edit server.port configuration in application.yml - Try a different port
  • Modify frontend port: Edit PORT configuration in tagtag-ui/apps/tagtag/.env - Frontend can also change ports
  • Find and terminate the process occupying the port - Whoever is occupying the port, kill it

5. Maven Dependency Download Failure

Problem Description: Dependency download failure when executing mvn clean package.

Solutions:

  • Check network connection - If the network is down, nothing can be downloaded
  • Clean Maven local repository: Delete ~/.m2/repository directory and re-download dependencies - Sometimes cache can have issues
  • Configure Maven mirror: Configure domestic mirror source in settings.xml - Domestic downloads are slow, use mirrors to accelerate

6. pnpm Dependency Installation Failure

Problem Description: Dependency installation failure when executing pnpm install.

Solutions:

  • Check network connection - Network issues, an old story
  • Clean pnpm cache: Execute pnpm store prune - Try clearing the cache
  • Upgrade pnpm version: Execute npm install -g pnpm - Using the latest version is never wrong
  • Configure pnpm mirror: Configure domestic mirror source in .npmrc - For domestic downloads, mirror acceleration is king

7. Compilation Error

Problem Description: Error during compilation.

Solutions:

  • Check if JDK version meets requirements (JDK 17+) - Don't use old versions anymore
  • Check if Node.js version meets requirements (Node.js >= 20.12.0) - Frontend also needs to keep up
  • Clean compilation cache: Execute mvn clean or pnpm run clean - Clear the cache
  • Check for syntax errors or type errors in code - You wrote the code, you check it

8. Login Failure

Problem Description: Login failure using initial account.

Solutions:

  • Confirm account and password are correct (initial account: admin / admin123) - Don't type them wrong
  • Check if backend service is running normally - If the backend isn't running, of course you can't log in
  • Check if user data exists in database - Was the database initialized?
  • Check JWT configuration - If JWT configuration is wrong, login will definitely fail
