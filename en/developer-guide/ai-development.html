AI-Assisted Development

Use AI and MCP to boost Tagtag development efficiency

Tagtag framework fully supports AI-assisted development through Model Context Protocol (MCP) integration and AI-friendly documentation, enabling you to leverage AI tools to accelerate your development workflow.

What is MCP?

Model Context Protocol (MCP) is an open standard for connecting AI assistants (like Claude, ChatGPT, etc.) to external data sources and tools. Through MCP, AI can directly access your project documentation, codebase, and other resources, providing more accurate and relevant assistance.

Tagtag's AI Support Features

1. AI-Friendly Documentation

Tagtag's official documentation has integrated the nuxt-llms module to automatically generate AI-readable documentation files:

  • llms.txt: Concise structured documentation optimized for AI to quickly understand the project
  • llms-full.txt: Complete detailed documentation providing in-depth project information

Access URLs:

  • http://tagtag.dev/llms.txt
  • http://tagtag.dev/llms-full.txt

These documents follow the llms.txt standard and can be directly read and parsed by various AI tools and MCP servers.

2. MCP Tool Integration

Tagtag documentation site has configured @nuxtjs/mcp-toolkit, providing standard MCP tool interfaces that allow AI assistants to directly access and query documentation content.

Using AI-Assisted Development in Trae

Step 1: Configure MCP Server

In Trae, you need to configure an MCP server to connect to Tagtag's documentation resources.

MCP Configuration Example

Create or edit your MCP configuration file (usually in the user configuration directory):

{
  "mcpServers": {
    "tagtag-docs": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "http://tagtag.dev/llms.txt"
      ],
      "env": {}
    }
  }
}

Step 2: Enable MCP in Trae

  1. Open Trae settings
  2. Navigate to "AI" or "MCP" configuration section
  3. Add Tagtag documentation MCP server configuration
  4. Save and restart Trae

Step 3: Use AI Assistant

Once configured, you can use the AI assistant in Trae:

AI will automatically access Tagtag documentation, understand the framework structure, and provide accurate code examples and guidance.

package dev.tagtag.module.product.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import dev.tagtag.module.product.dto.ProductDTO;
import dev.tagtag.module.product.dto.ProductQuery;
import dev.tagtag.module.product.entity.Product;
import dev.tagtag.module.product.mapper.ProductMapper;
import dev.tagtag.module.product.service.ProductService;
import dev.tagtag.common.core.page.PageResult;
import dev.tagtag.common.core.exception.BusinessException;
import lombok.RequiredArgsConstructor;
import org.springframework.beans.BeanUtils;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
public class ProductServiceImpl implements ProductService {
    
    private final ProductMapper productMapper;
    
    @Override
    public PageResult<ProductDTO> pageProducts(ProductQuery query) {
        LambdaQueryWrapper<Product> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(query.getName() != null, Product::getName, query.getName())
               .eq(query.getCode() != null, Product::getCode, query.getCode())
               .eq(query.getStatus() != null, Product::getStatus, query.getStatus())
               .orderByDesc(Product::getCreateTime);
        
        Page<Product> page = productMapper.selectPage(
            new Page<>(query.getPageNum(), query.getPageSize()), 
            wrapper
        );
        
        return PageResult.of(page, this::convertToDTO);
    }
    
    @Override
    public ProductDTO getProductById(Long id) {
        Product product = productMapper.selectById(id);
        if (product == null) {
            throw new BusinessException("Product does not exist");
        }
        return convertToDTO(product);
    }
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void createProduct(ProductDTO productDTO) {
        Product product = convertToEntity(productDTO);
        productMapper.insert(product);
    }
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void updateProduct(ProductDTO productDTO) {
        Product product = productMapper.selectById(productDTO.getId());
        if (product == null) {
            throw new BusinessException("Product does not exist");
        }
        BeanUtils.copyProperties(productDTO, product);
        productMapper.updateById(product);
    }
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void deleteProduct(Long id) {
        productMapper.deleteById(id);
    }
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void batchDeleteProducts(java.util.List<Long> ids) {
        ids.forEach(this::deleteProduct);
    }
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void updateProductStatus(Long id, Integer status) {
        Product product = productMapper.selectById(id);
        if (product == null) {
            throw new BusinessException("Product does not exist");
        }
        product.setStatus(status);
        productMapper.updateById(product);
    }
    
    private ProductDTO convertToDTO(Product product) {
        ProductDTO dto = new ProductDTO();
        BeanUtils.copyProperties(product, dto);
        return dto;
    }
    
    private Product convertToEntity(ProductDTO dto) {
        Product product = new Product();
        BeanUtils.copyProperties(dto, product);
        return product;
    }
}

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/products")
@RequiredArgsConstructor
public class ProductController {
    
    private final ProductService productService;
    
    @GetMapping
    public Result<PageResult<ProductDTO>> pageProducts(ProductQuery query) {
        PageResult<ProductDTO> result = productService.pageProducts(query);
        return Result.success(result);
    }
    
    @GetMapping("/{id}")
    public Result<ProductDTO> getProductById(@PathVariable Long id) {
        ProductDTO product = productService.getProductById(id);
        return Result.success(product);
    }
    
    @PostMapping
    public Result<Void> createProduct(@RequestBody ProductDTO productDTO) {
        productService.createProduct(productDTO);
        return Result.success();
    }
    
    @PutMapping
    public Result<Void> updateProduct(@RequestBody ProductDTO productDTO) {
        productService.updateProduct(productDTO);
        return Result.success();
    }
    
    @DeleteMapping("/{id}")
    public Result<Void> deleteProduct(@PathVariable Long id) {
        productService.deleteProduct(id);
        return Result.success();
    }
    
    @DeleteMapping("/batch")
    public Result<Void> batchDeleteProducts(@RequestBody java.util.List<Long> ids) {
        productService.batchDeleteProducts(ids);
        return Result.success();
    }
    
    @PutMapping("/{id}/status")
    public Result<Void> updateProductStatus(@PathVariable Long id, @RequestParam Integer status) {
        productService.updateProductStatus(id, status);
        return Result.success();

Tagtag adopts an RBAC (Role-Based Access Control) permission model, including the following core concepts:

  • User: System users
  • Role: Collection of permissions, users gain permissions through roles
  • Permission: Operation permissions on resources
  • Menu: Frontend menu and button permission control

3. Permission Validation Implementation

// Permission validation annotation
package dev.tagtag.common.security.annotation;

import java.lang.annotation.*;

@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface RequiresPermissions {
    
    String[] value() default {};
    
    Logical logical() default Logical.AND;
    
    enum Logical {
        AND, OR
    }
}

// Permission validation aspect
package dev.tagtag.common.security.aspect;

import dev.tagtag.common.security.annotation.RequiresPermissions;
import dev.tagtag.common.security.service.PermissionService;
import dev.tagtag.common.core.exception.ForbiddenException;
import lombok.RequiredArgsConstructor;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.reflect.MethodSignature;
import org.springframework.stereotype.Component;

@Aspect
@Component
@RequiredArgsConstructor
public class PermissionAspect {
    
    private final PermissionService permissionService;
    
    @Before("@annotation(dev.tagtag.common.security.annotation.RequiresPermissions)")
    public void checkPermission(JoinPoint joinPoint) {
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        RequiresPermissions requiresPermissions = signature.getMethod()
            .getAnnotation(RequiresPermissions.class);
        
        String[] permissions = requiresPermissions.value();
        RequiresPermissions.Logical logical = requiresPermissions.logical();
        
        boolean hasPermission = permissionService.hasPermissions(permissions, logical);
        
        if (!hasPermission) {
            throw new ForbiddenException("Insufficient permissions");
        }
    }
}

// Permission service
package dev.tagtag.common.security.service;

import dev.tagtag.common.security.utils.SecurityUtils;
import dev.tagtag.module.iam.mapper.UserMapper;
import dev.tagtag.module.iam.mapper.RoleMapper;
import dev.tagtag.module.iam.mapper.PermissionMapper;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.Set;
import java.util.stream.Collectors;

@Service
@RequiredArgsConstructor
public class PermissionService {
    
    private final UserMapper userMapper;
    private final RoleMapper roleMapper;
    private final PermissionMapper permissionMapper;
    
    public boolean hasPermissions(String[] permissions, RequiresPermissions.Logical logical) {
        Long userId = SecurityUtils.getUserId();
        Set<String> userPermissions = getUserPermissions(userId);
        
        if (logical == RequiresPermissions.Logical.AND) {
            return userPermissions.containsAll(List.of(permissions));
        } else {
            return List.of(permissions).stream()
                .anyMatch(userPermissions::contains);
        }
    }
    
    private Set<String> getUserPermissions(Long userId) {
        List<Long> roleIds = userMapper.getRoleIdsByUserId(userId);
        return roleIds.stream()
            .flatMap(roleId -> permissionMapper.getPermissionCodesByRoleId(roleId).stream())
            .collect(Collectors.toSet());
    }
}

// Permission directive
import type { Directive, DirectiveBinding } from 'vue';

const permission: Directive = {
  mounted(el: HTMLElement, binding: DirectiveBinding) {
    const { value } = binding;
    const permissions = useUserStore().permissions;
    
    if (value && !permissions.includes(value)) {
      el.parentNode?.removeChild(el);
    }
  }
};

export default permission;

// Permission composable function
import { computed } from 'vue';
import { useUserStore } from '@/store/user';

export function usePermission() {
  const userStore = useUserStore();
  
  const hasPermission = (permission: string | string[]): boolean => {
    const permissions = userStore.permissions;
    if (Array.isArray(permission)) {
      return permission.some(p => permissions.includes(p));
    }
    return permissions.includes(permission);
  };
  
  const hasAllPermissions = (permissions: string[]): boolean => {
    const userPermissions = userStore.permissions;
    return permissions.every(p => userPermissions.includes(p));
  };
  
  const hasAnyPermission = (permissions: string[]): boolean => {
    const userPermissions = userStore.permissions;
    return permissions.some(p => userPermissions.includes(p));
  };
  
  return {
    hasPermission,
    hasAllPermissions,
    hasAnyPermission
  };
}

5. Backend Permission Annotation Usage

// Use permission annotations in Controller
package dev.tagtag.module.iam.controller;

import dev.tagtag.common.security.annotation.RequiresPermissions;
import dev.tagtag.common.core.result.Result;
import dev.tagtag.module.iam.dto.UserDTO;
import dev.tagtag.module.iam.service.UserService;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/users")
@RequiredArgsConstructor
public class UserController {
    
    private final UserService userService;
    
    @GetMapping
    @RequiresPermissions("user:list")
    public Result<PageResult<UserDTO>> pageUsers(UserQuery query) {
        PageResult<UserDTO> result = userService.pageUsers(query);
        return Result.success(result);
    }
    
    @GetMapping("/{id}")
    @RequiresPermissions("user:query")
    public Result<UserDTO> getUserById(@PathVariable Long id) {
        UserDTO user = userService.getUserById(id);
        return Result.success(user);
    }
    
    @PostMapping
    @RequiresPermissions("user:create")
    public Result<Void> createUser(@RequestBody UserDTO userDTO) {
        userService.createUser(userDTO);
        return Result.success();
    }
    
    @PutMapping
    @RequiresPermissions("user:update")
    public Result<Void> updateUser(@RequestBody UserDTO userDTO) {
        userService.updateUser(userDTO);
        return Result.success();
    }
    
    @DeleteMapping("/{id}")
    @RequiresPermissions("user:delete")
    public Result<Void> deleteUser(@PathVariable Long id) {
        userService.deleteUser(id);
        return Result.success();
    }
    
    @PutMapping("/{id}/status")
    @RequiresPermissions(value = {"user:update", "user:status"}, logical = Logical.OR)
    public Result<Void> updateUserStatus(@PathVariable Long id, @RequestParam Integer status) {
        userService.updateUserStatus(id, status);
        return Result.success();
    }
}

Best Practices

1. Provide Clear Context

When asking AI questions, provide sufficient context information:

// Good question
"In the Tagtag framework, I need to create a new module 'order' with order management functionality.
Please reference the existing 'user' module structure and generate corresponding code."

"Help me write an order module"

// Step 1
"Help me create an Order entity class with fields like id, userId, amount, status"

// Step 2
"Based on the Order entity, create the corresponding Mapper interface"

// Step 3
"Create OrderService interface and implementation class"

FAQ

Q: Can I use AI-generated code directly?

A: Not recommended. AI-generated code needs to be manually reviewed, tested, and optimized to ensure it meets project standards and business requirements.

Q: How can I improve AI response quality?

A: Provide clear, specific context information, reference Tagtag documentation structure, and use framework terminology. This way, AI can provide more accurate answers.

Q: What should I do if MCP server connection fails?

A: Check network connection, ensure documentation URL is accessible, review MCP server logs, and verify configuration is correct.

Q: Can I use AI-assisted development offline?

A: Yes. You can download the llms.txt file locally, configure a local MCP server, and achieve offline access.
